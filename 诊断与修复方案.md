# v1.1.0 改进方案（详细）

---

## 1. 错题本/收藏题库 题目不截断

**现状：** 错题本和收藏题库的题目卡片中，`QuestionTable.tsx` 用 `truncate` 或单行显示长文本，导致长题目被截断

**修复：**
- `QuestionTable.tsx`：题目列加 `whitespace-pre-wrap` + 去掉 `truncate`，让长题目自动换行全部显示
- `WrongBook/index.tsx`：题目卡片同样去掉截断

**涉及文件：** `src/components/shared/QuestionTable.tsx`、`src/components/WrongBook/index.tsx`、`src/components/FavoriteBank/index.tsx`

---

## 2. 刷题界面导航栏优化（4 个页面统一）

**现状：** 刷题、Review、错题重刷、收藏刷题这 4 个页面共享同一套布局，题目区在上，选项区在下，导航栏在选项区下面。题数多时导航栏是一长条网格，需要滚到页面底部才能点下一题

**修复（三个改动合在一起）：**

### 2a. 题目导航栏加滚动条
- `QuestionNav.tsx` 外层包一个 `div`，加 `max-h-[40vh] overflow-y-auto`
- 题目网格超出时出现滚动条，不是无限撑高页面

### 2b. 底部固定导航栏
- 在题目区域下方加一个**固定底部栏**（`sticky bottom-0`）
- 内容：左侧显示 `已刷 X / N 题`，中间显示快捷键提示 `← 上一题  Space 提交  → 下一题  F 标记`，右侧显示 `上一题` 和 `下一题` 按钮
- 这 4 个页面都要有（刷题、Review、错题重刷、收藏刷题）

### 2c. 电脑端不滚动
- 题目区 + 导航栏高度不超过一屏，电脑端不需要滚动即可同时看到题目和导航

**涉及文件：** `src/components/Practice/QuestionNav.tsx`、`src/components/Practice/index.tsx`

---

## 3. 快捷键

**覆盖范围：** 刷题、Review、错题重刷、收藏题库刷题 4 个页面

| 快捷键 | 功能 |
|--------|------|
| `←` 或 `A` | 上一题 |
| `→` 或 `D` | 下一题 |
| `Space` | 提交答案（刷题时）/ 查看答案（review 时） |
| `F` | 标记 / 取消标记当前题目 |
| `1`-`9` | 直接选择对应选项（仅单选/多选/判断） |

**实现方式：** 在现有的 `useKeyboardShortcuts.ts` 中补充，监听 `keydown` 事件

**涉及文件：** `src/hooks/useKeyboardShortcuts.ts`

**注意：** 快捷键只在刷题/Review 阶段生效，配置阶段不生效（用户在输入框里打字时也不生效）

## 4. Review 添加标记和收藏同步

**现状：** Review 界面的题目卡片上只有"已查看答案"标识，没有 Flag 标记按钮

**修复：**
- Review 题目卡片上加 Flag 图标按钮（和刷题时一样的位置）
- 点击切换标记 → 调用 `markQuestion()` → 同时同步到 `useConfigStore` 的 `favorites`
- 已标记的题目显示橙色/黄色 Flag，未标记的显示灰色

**涉及文件：** `src/components/Practice/index.tsx`（Review 渲染部分，约 1000-1080 行）

## 5. 清空所有数据

**现状：** Settings 没有全清按钮。现有的 `clearAll` 只清 4 个 key，而且没有确认弹窗

**修复：**
- Settings 页面加"清空所有数据"按钮（红色警示色，放在页面底部）
- 点击 → `ConfirmDialog`：文案 "确定清空所有数据吗？题库、错题、练习记录、配置、AI 设置全部删除，不可恢复！"
- 确认 → `storage.clearAll()` + 清空所有 store（`useBankStore`、`useWrongStore`、`usePracticeStore`、`useConfigStore`）→ 跳转回首页
- `storage.clearAll()` 需确认清空了全部 localStorage key：`quiz_banks`、`quiz_wrong`、`quiz_records`、`quiz_config`、`ai_config`、`ai_explanations`

**涉及文件：** `src/components/Settings/index.tsx`、`src/utils/storage.ts`

## 6. 清空标记拆成两个按钮

**现状：** BankDetail 只有一个"清空标记"按钮，调用 `clearBankWrongs()`

**修复：** 拆成两个按钮，文案区分清楚：

**涉及文件：** `src/components/Bank/BankDetail.tsx`

## 8. 页面缓存 Bug（重点）

**现状复现：**
1. 打开错题重刷 → 配置完 → 出现题目（phase = 'practice'）
2. 点击导航栏题库广场 → 点多题库刷题 → 开始刷题
3. **Bug：** 没有进入多题库配置页，直接进了之前错题重刷的刷题界面
4. 原因：`phase` 存在 zustand store，上一个 mode 的 `practice` 阶段残留了

**修复：**
- `App.tsx` 中所有进入刷题的入口（`handleStartPractice`、`handleWrongReview`、`handleMultiBankPractice`、`handleFavoritePractice`）在调用 `setCurrentPage('practice')` 之前，先调用 `usePracticeStore.getState().resetPractice()`
- `Practice/index.tsx`：当 `mode` 改变时（用 `useEffect` 监听），自动 `resetPractice()`

**涉及文件：** `src/App.tsx`、`src/components/Practice/index.tsx`

---

## 9. 返回按钮不受影响

**分析：** 返回按钮依赖 `App.tsx` 中的 `navHistory` 栈和 `handleNavBack`，与 practice store 独立，互不影响。修复 #8 不会破坏返回按钮

**验证：** 修复后测试场景：刷题中 → 点导航栏其他页 → 点返回 → 回到刷题当前题目

## 10. AI 提示词模板（独立功能，后做）

**概念：**
- 3 个模式：刷题（practice）、Review、分析（analyze）
- 每个模式 3 个模板，共 9 个
- 每个模板有 `名称` 和 `内容`，均可编辑
- 每个模式可选择当前使用的模板
- 清空数据时重置为默认

**类型定义：**
```ts
interface AIPromptTemplate {
  id: string;
  name: string;
  content: string;
}

interface AIPromptTemplates {
  practice: AIPromptTemplate[];   // 3 个
  review: AIPromptTemplate[];     // 3 个
  analyze: AIPromptTemplate[];    // 3 个
}

interface AIPromptConfig {
  templates: AIPromptTemplates;
  activeIds: {
    practice: string;
    review: string;
    analyze: string;
  };
}
```

**默认模板：** 内置 9 个默认模板（中英文各 3 套），用户可改

**存储：** `localStorage` 的 `quiz_config` 中或独立 key `ai_prompts`

**UI：** AI 配置文件页面新增"提示词模板"选项卡，展示 3 个模式 × 3 个模板的卡片网格，每个卡片可编辑名称和内容，可选择"使用此模板"

**涉及文件：** `src/types/index.ts`、`src/utils/ai.ts`、`src/components/AI/AIConfigPage.tsx`、`src/components/Settings/index.tsx`
