# 更新日志空白修复方案 v2

## 问题

网页版 fetch Gitee 被 CORS 拦截 → 静默失败 → changelog 空数组 → 空白。

Tauri 里不受 CORS 限制，能拿到数据。但网页版不行。

## 方案：动态 + 兜底

保持 fetch 逻辑，但加一个**硬编码兜底**：

```tsx
const FALLBACK_CHANGELOG: ChangelogEntry[] = [
  { version: '1.0.4', items: [...] },
  { version: '1.0.3', items: [...] },
  ...
];

// fetch 成功 → 用 parsed
// fetch 失败 → 用 FALLBACK_CHANGELOG
```

fetch 成功后 `setChangelog(parseNotes(...))`；catch 里 `setChangelog(FALLBACK_CHANGELOG)`。

## 优点

- 网页版永远有内容（兜底）
- Tauri/生产环境自动更新
- 每次发版只需更新 version.json 和兜底数组（写 deploy.bat 自动化）

## 改动

| 文件 | 改动 |
|------|------|
| `src/components/Settings/index.tsx` | 加 FALLBACK_CHANGELOG 兜底 |

## 展望

以后发版可以写脚本自动更新兜底数组，但现阶段手动改两处也能接受，因为兜底只是网页版用。
