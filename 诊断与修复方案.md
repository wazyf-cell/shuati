# 四项改进方案

## 一、简答题导入不完整

### 问题
Excel 导入简答题时，`parseCorrectAnswer` 只取原始文本作为 `correctAnswer[0]`，完全忽略 `subType` 和 `subQuestions`，导致大题（group）结构丢失。

TXT 导入正常（`[简答-大题]` 标签支持 group）。

### 修复
Excel 解析器加简答 group 支持：检测 `|` 分隔的"小题:答案"格式，自动解析为 subQuestions。

| 文件 | 改动 |
|------|------|
| `src/utils/excelParser.ts` | 简答题列解析支持 group subQuestions |

---

## 二、简答题 AI 不扫描选项

### 问题
`generateExplanation` 的 prompt 构建只遍历 `question.options`，但简答题 `options = []`，导致 `subQuestions` 数据完全没传给 AI。

### 修复
在 prompt 中加简答题专属段：如果 `type === 'short' && subQuestions?.length > 0`，追加小题列表。

| 文件 | 改动 |
|------|------|
| `src/utils/ai.ts` | `generateExplanation` 加 subQuestions 输出 |

---

## 三、简答题编辑时选项和答案中间的分割线可左右拖动

### 问题
QuestionForm 中选项区和答案区是纵向排列，没有左右分栏。用户要的是左右布局，中间分割线可拖拽。

### 修复
`type === 'short'` 时改为左右两栏布局：
- 左栏：题目 + 选项（如有）
- 中：可拖拽分割线（`resize` cursor）
- 右栏：答案区（single 输入 / group subQuestions）

| 文件 | 改动 |
|------|------|
| `src/components/Bank/QuestionForm.tsx` | short 类型时左右分栏 + 可拖拽分割线 |

---

## 四、三层题库结构（文件夹+题库）

### 现状
题库是扁平列表 `QuestionBank[]`，没有层级。

### 方案
最小改动方案：给 `QuestionBank` 加可选 `parentId`，Dashboard 按文件夹分组渲染。

**类型扩展：**
```ts
interface QuestionBank {
  id: string;
  name: string;
  parentId?: string;    // null/undefined = 根目录
  questions: Question[];
  createdAt: number;
  updatedAt: number;
}

interface BankFolder {
  id: string;
  name: string;
  parentId?: string;     // 支持嵌套，最多3层
  order: number;
}
```

**存储：**
- `quiz_banks` 不变（parentId 可选，旧数据兼容）
- 新建 `quiz_folders` 存文件夹

**UI：**
- Dashboard 不再展示扁平的卡片列表
- 改为树形结构：文件夹可折叠展开，题库在文件夹内
- 根目录可直接创建题库（兼容旧版）
- 右键菜单或拖拽移动题库到文件夹

**涉及文件：**

| 文件 | 改动 |
|------|------|
| `src/types/index.ts` | 加 `BankFolder` 类型，`QuestionBank` 加 `parentId?` |
| `src/store/bank.ts` | 加 folder CRUD、移动题库 |
| `src/utils/storage.ts` | 加 folders 读写 |
| `src/components/Dashboard/index.tsx` | 扁平列表 → 树形 |
| `src/components/Dashboard/BankCard.tsx` | 适配 |
| `src/components/Dashboard/FolderNode.tsx` | **新建**，文件夹节点 |

### 数据迁移
自动兼容：旧题库 `parentId` 缺失 → 视为根目录，显示无变化。
