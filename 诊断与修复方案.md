# AI 解析保存 + 与题目解析共存方案

## 现状分析

| 数据类型 | 存储位置 | 何时生成 | 内容 |
|---------|---------|---------|------|
| `question.analysis` | 题库数据（localStorage） | 手动写 / 编辑时 AI 生成 | 通用解析，和答案无关 |
| `AIExplanationCard` 结果 | `localStorage` key `ai_explanations` | 刷题时点 AI 按钮 | 针对用户错误答案的个性化解析 |

两者**已经各自独立保存**，不需要额外开发"保存"功能。

## 共存方案

### 刷题时的展示顺序（从上到下）

```
┌─ 题目 ─────────────────┐
│  题干                    │
│  选项                    │
├─ 查看正确答案后 ─────────┤
│  正确答案                 │
│  💡 题目解析（question.analysis，有则显示）│
│  🤖 AI 解析本题（AIExplanationCard，按钮/结果）│

```

```
┌─ Review 每题 ───────────┐
│  题型标签 + 题目 + 选项   │
│  💡 解析（question.analysis，有则显示）│
│  🤖 AI 解析（AIExplanationCard，仅错题）│
└─────────────────────────┘
```

### 编辑页

```
┌─ QuestionForm ──────────┐
│  题目解析 textarea        │  ← 手动写
│  🤖 AI 生成解析 按钮      │  ← 点后自动填入并保存
└─────────────────────────┘
```

## 题目解析输入框自适应高度

### 现状
`QuestionForm.tsx` 解析 textarea 固定高度 + 手动拖拽 resize-y + 滚动条。

### 改为
根据内容自动撑开高度，无滚动条，无手动拖拽手柄。

```tsx
// 用 useRef + useEffect 监听 analysis 变化
const textareaRef = useRef<HTMLTextAreaElement>(null);

useEffect(() => {
  const el = textareaRef.current;
  if (el) {
    el.style.height = 'auto';
    el.style.height = el.scrollHeight + 'px';
  }
}, [analysis]);

<textarea
  ref={textareaRef}
  value={analysis}
  onChange={(e) => setAnalysis(e.target.value)}
  placeholder="输入题目解析..."
  className="input min-h-[60px] overflow-hidden"
  rows={1}
/>
```

### 效果
- 手动输入时逐行自动增长
- AI 生成解析填入后自动撑开到完整高度
- 不需要拖拽手柄，也不需要滚动条

| 文件 | 改动 |
|------|------|
| `src/components/Bank/QuestionForm.tsx` | textarea 改自适应 |

## 总结

| 问题 | 答案 |
|------|------|
| AI 解析保存了吗 | ✅ 生成即保存（编辑时写 question.analysis，刷题时写 localStorage） |

| 需要新增功能吗 | 不需要，已在完整体现 |
