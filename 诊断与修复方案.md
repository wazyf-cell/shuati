# 最终修复方案

## 根因确认

| 请求方式 | GitHub Raw 结果 |
|---------|---------------|
| 浏览器 | ✅ 200，正常 |
| Tauri WebView2 fetch() | ❌ 超时/不通 |
| Rust reqwest | ❌ 404 |

**结论：Tauri 网络栈无法访问 GitHub Raw，只有浏览器自带代理可以。**

## 修复方案

### 一、版本检测 → 切回 Gitee（之前 reqwest 能正常访问）

```
UPDATE_URL = https://gitee.com/zhong-yongfu/shuati/raw/master/shuati-update/version.json
```

Gitee 对文本文件（version.json）不拦截，之前 Rust reqwest 能正常返回 200。

### 二、下载文件 → 用浏览器打开（open_url 命令）

```rust
// download_update 失败时，前端 fallback 到 open_url
// 浏览器下载后用户手动替换（这是回退方案）
```

### 三、错误展示 → 改成红色醒目标签

```tsx
// 三种状态各自独立，用醒目颜色：
// 网络错误 → 红色标签（大号字体）
// 已是最新 → 绿色标签
// 有新版本 → 橙色链接
```

### 四、404 永不再出现

改用 Gitee URL 做检测，Rust reqwest 已验证能通。
