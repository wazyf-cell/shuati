# 版本号 + 更新日志方案

## 问题
- 版本号要改 1.0.3
- 设置页更新弹窗只显示最新版公告，看不到之前版本的更新内容

## 方案

### 一、版本升级到 1.0.3

| 文件 | 改动 |
|------|------|
| `src/version.ts` | `1.0.2` → `1.0.3` |
| `package.json` | `1.0.2` → `1.0.3` |
| `src-tauri/tauri.conf.json` | `1.0.2` → `1.0.3` |

### 二、更新日志累计显示

`gitee-update/version.json` 的 `notes` 字段改为**累计更新日志**，包含所有历史版本：

```json
{
  "version": "1.0.3",
  "downloadUrl": "...",
  "apkDownloadUrl": "...",
  "notes": "v1.0.3\n修复版本号不统一问题\n\nv1.0.2\n新增收藏题库\n新增自动加入错题本开关\n新增题目编辑 AI 生成解析\n新增题库清空错题标记\nAI 配置前往官网自动打开浏览器\n优化导入弹窗居中\n\nv1.0.1\n新增 AI 配置\n错题本折叠\n多题库按题型分类\n修复导航 Key 警告\n修复导入导出图标互换"
}
```

### 三、显示效果

检测到 1.0.3 时，弹窗/设置页显示的更新内容包含 1.0.3 + 1.0.2 + 1.0.1 的所有条目，用户不会遗漏中间版本的更新。

### 不变的部分
代码逻辑不改，只是 notes 内容变长 —— 现有展示逻辑直接支持多行文本（`whitespace-pre-wrap`）。
