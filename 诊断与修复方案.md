# Android 修复方案

## 问题 1：Failed to fetch

**原因**：Android WebView 内 `fetch()` 到 Gitee 超时/不通。和 Tauri 一样——国内网络下，WebView 不能稳定访问 raw Git 服务。

**修复**：改用 Capacitor HTTP Plugin，或者直接让用户手动点"检查更新"。

> 实际可行的方案：Android 启动后不做自动检测（静默），用户进设置页手动点"检查更新"时才发请求。

## 问题 2：下载的是 zip 不是 apk

**原因**：Gitee Raw URL 不能直接提供 APK 二进制下载（和 exe 一样被拦截），返回的是网页或 zip。

**修复**：把 APK 放到不拦截的服务上：
- **方案 A**：GitHub Raw（浏览器能下载，Android WebView 不能自动检测但浏览器打开可以）
- **方案 B**：把 APK 放 Gitee Release 页（让用户浏览器打开下载）

**方案 A** 更好，`apkDownloadUrl` 改成 GitHub Raw，浏览器打开就能直接下载 APK。
