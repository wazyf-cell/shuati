# 部署与更新说明

## 一、项目结构

```
program6-shuati/
├── src-tauri/           ← Tauri 桌面端代码（Rust）
├── src/                 ← 前端代码（React + Vite）
├── versel-update/       ← Vercel 更新文件目录
│   ├── version.json     ← 版本信息
│   └── shuati_*.exe     ← 安装包
└── dist/                ← 构建产物
```

## 二、打包桌面应用

```bash
npm run tauri:build
```

产出物：

| 用途 | 文件 | 路径 |
|------|------|------|
| **绿色版（发给朋友）** | `shuati.exe` | `src-tauri/target/release/shuati.exe` |
| 安装包 | `shuati_0.1.0_x64-setup.exe` | `src-tauri/target/release/bundle/nsis/` |

**绿色版** 双击即用，无需安装，推荐直接分享。

## 三、发布新版（每次更新时）

### 完整步骤

**1. 改版本号**

`src-tauri/tauri.conf.json` 中的 `version` 字段（如改成 `0.2.0`）

**2. 打包**

```bash
npm run tauri:build
```

**3. 更新 version.json**

编辑 `versel-update/version.json`：

```json
{
  "version": "0.2.0",
  "downloadUrl": "https://versel-update.vercel.app/shuati.exe",
  "notes": "新功能说明"
}
```

> `downloadUrl` 始终指向 `shuati.exe`（绿色版），不写版本号，这样每次覆盖上传即可，链接不变。

**4. 复制新 exe + 部署到 Vercel**

```bash
copy src-tauri\target\release\shuati.exe versel-update\
cd versel-update
vercel deploy --prod
```

**5. 更新代码中的 APP_VERSION**

`src/components/Settings/index.tsx` 中的 `APP_VERSION` 常量改为新版本号。

### 总结：每次发布改 3 处

| 改什么 | 位置 |
|--------|------|
| 版本号 | `src-tauri/tauri.conf.json` |
| 版本号 | `src/components/Settings/index.tsx` 的 `APP_VERSION` |
| 版本号 + notes | `versel-update/version.json` |

## 四、自动更新流程

```
用户打开设置 → 检查更新
    ↓
请求 https://versel-update.vercel.app/version.json
    ↓
对比 version ≠ APP_VERSION
    ↓
弹出通知 → 点"自动更新" → 下载 shuati.exe → 替换旧文件
```

## 五、注意事项

- `version.json` 和 Settings 页面 `APP_VERSION` 必须一致
- 下载链接始终指向 `shuati.exe`（绿色版），不写版本号
- 导出数据包含 AI 配置（含 API Key），重置后不会包含
- Vercel 别名固定 `https://versel-update.vercel.app`，每次 `--prod` 自动更新
